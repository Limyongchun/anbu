[ANBU 앱 디자인 시스템 & UI 리디자인 바이브코딩 지시서]

목표:
기존 기능은 유지하면서,
“노부모 케어 앱”을 “프리미엄 라이프케어 플랫폼”처럼 보이도록
디자인 시스템 기반으로 전면 리디자인한다.

레퍼런스 스타일:
- KakaoBank → 카드 기반, 컬러 블록, 정보 명확성
- Runna → 데이터 카드 + 상태 표현 + 좌측 컬러 인디케이터
- Stoic → 여백, 타이포, 부드러운 그라데이션
- Forest → 지도 + 플로팅 UI 구조

5. 핵심 컴포넌트 정의
--------------------------------

[1] App Header
- 좌측: ANBU 로고
- 우측: 상태 배지 (마스터)
- 배경: transparent or soft blur

[2] Status Chip
- rounded pill
- icon + text
- 색상: status 기반

[3] Parent Card (핵심 컴포넌트)
구조:
- 좌측 컬러 인디케이터 (Runna 스타일)
- 프로필 + 이름
- 상태 chip (활동중 / 비활동)
- 마지막 활동 시간
- 하단: activity stats (아이콘 + 숫자)

variant:
- normal
- warning (amber)
- safe (green)

[4] Feed Card (사진/메시지)
- 상단: 사용자 + 시간
- 이미지 영역
- 하단: 액션 (좋아요 / 삭제)
- 강조는 border 또는 soft background만

👉 형광 라임 배경 카드 사용 금지

[5] Floating Action Button
- 원형
- 라임 색상
- shadow strong
- 위치: 하단 우측

[6] Bottom Tab Bar
- 아이콘 only + label
- active: lime
- inactive: gray

6. 화면별 리디자인 지시
--------------------------------

[홈 화면]

구조:
1. Today Summary Card (신규 추가)
- "오늘 부모님 상태"
- 부모 2명 상태 요약
- 문제 있으면 강조

2. Parent Cards (2개)
- Runna 스타일 적용
- 좌측 컬러 라인 필수

3. Recent Activity
- 타임라인 형태
- 카드 간 여백 크게

👉 목표:
“정보판 느낌”으로 재구성

--------------------------------

[사진 화면]

금지:

추가:
- 상단 segment (메시지 / 사진)

7. 모션 시스템
--------------------------------

기본:
- duration: 180~220ms
- easing: ease-in-out

애니메이션:
- 카드 등장: fade + translateY(12)
- 탭 전환: fade
- 모달: scale 0.98 → 1
- 상태 변경: color transition

금지:
- 과한 bounce
- 긴 애니메이션

--------------------------------
8. UX 핵심 원칙
--------------------------------

- 한 화면 = 핵심 메시지 1개
- 텍스트 대신 상태 + 숫자 + 아이콘
- 여백은 “과하다 싶게”
- 카드 역할은 반드시 구분
- 모든 UI는 "부모 안심"을 위해 존재
